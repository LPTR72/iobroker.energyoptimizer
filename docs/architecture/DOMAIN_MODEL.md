# Domain model

The domain model describes the home energy system in a vendor-neutral way.

## Core ideas

- Energy assets represent physical energy behavior rather than concrete brands or protocols.
- Measurements, forecasts, predictions, evaluations, recommendations, and later plans use neutral contracts.
- ioBroker object IDs, adapter lifecycle APIs, cloud APIs, and transport protocols remain integration details.

## Important model areas

- Energy assets
- Energy system state
- Optimizer input
- Analysis
- Forecasts
- Predictions
- Energy situations
- Recommendations
- Planning models
- Historical metrics and buckets
- Pattern hypotheses and future virtual assets

## Pattern-based knowledge

Future Pattern-based Virtual Energy Assets are confirmed knowledge derived from historical behavior. They are not automatic device identification. Detected regularity remains a hypothesis until the user confirms it.
