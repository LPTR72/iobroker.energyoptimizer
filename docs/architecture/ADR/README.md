# Architecture Decision Records

Architecture Decision Records (ADRs) capture important technical choices and their consequences. Accepted ADRs describe the current direction; they do not imply that every planned downstream component is implemented.

- [ADR-0001: Clean Architecture](ADR-0001-clean-architecture.md)
- [ADR-0002: Generic asset model](ADR-0002-generic-asset-model.md)
- [ADR-0003: Analysis engine](ADR-0003-analysis-engine.md)
- [ADR-0004: Forecast abstraction](ADR-0004-forecast-abstraction.md)
- [ADR-0005: Prediction engine](ADR-0005-prediction-engine.md)
- [ADR-0006: Evaluation engine](ADR-0006-evaluation-engine.md)

New ADRs should be numbered sequentially and contain Status, Context, Decision, and Consequences sections.
