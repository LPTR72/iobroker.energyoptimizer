# Roadmap

This roadmap summarizes the public project direction.

## Completed foundations

- Adapter lifecycle and safe polling
- Live value mirroring
- Fixed-tariff import-cost calculation
- Generic energy assets and normalized configuration
- Analysis, forecast abstraction, prediction, evaluation, and recommendation foundations
- Read-only simulation runtime integration
- Structured read-only recommendation output
- Dormant planning semantics

## Current work

The current work focuses on the History Service domain foundation and its architecture review.

## Planned next areas

- History Service runtime integration
- Repository integration through existing ioBroker SQL infrastructure
- Forecast, tariff, and weather providers
- Pattern Recognition Engine
- Pattern-based Virtual Energy Assets / Knowledge Model
- Simulation Framework
- Separately approved integration milestones

## Long-term direction

The project aims to move gradually from transparent energy-data handling toward explainable optimization based on measurements, forecasts, history, patterns, priorities, and user-confirmed knowledge.
